# RAG система для технических заданий

Проект по заданию "КТ 11. Генерация с RAG по существующему ТЗ"

## Функционал

### 1. Разбиение ТЗ на блоки
Поддержка форматов:
- Markdown (# заголовки, ## подзаголовки, списки)
- Простой формат с разделителями (Раздел|Содержание)
- Обычный структурированный текст
- Нумерованные и маркированные списки

### 2. Векторизация и сохранение
- Создание 10-мерных векторных представлений текста
- Сохранение векторов в файл vectors.json
- Загрузка векторов из файла для повторного использования

### 3. Семантический поиск (RAG)
- Поиск по смыслу, а не только по ключевым словам
- Определение темы запроса (оплата, доставка, интерфейс и т.д.)
- Ранжирование результатов по релевантности
- Показ 5 наиболее релевантных результатов с возможностью просмотра остальных

## Как запустить проект

### Требования
- Windows 10/11
- .NET 8.0 SDK
- Visual Studio 2022 (или другая IDE для C#)

### Установка
1. Клонируйте репозиторий
2. Откройте решение `CoffeeBotRAG.sln` в Visual Studio
3. Нажмите F5 для сборки и запуска

### Альтернативный запуск через командную строку:
```
dotnet build
dotnet run
```

## Как использовать программу

### Шаг 1: Загрузите ТЗ
Нажмите "Загрузить ТЗ" и выберите текстовый файл (.txt или .md) с техническим заданием.

### Шаг 2: Векторизуйте текст
Нажмите "Векторизовать" для создания семантических векторов. Процесс занимает несколько секунд.

### Шаг 3: Сохраните векторы (опционально)
Нажмите "Сохранить векторы" чтобы сохранить результат в файл для будущего использования.

### Шаг 4: Выполните поиск
Введите вопрос в поле ввода и нажмите "Найти". Программа найдет наиболее релевантные блоки ТЗ.



## Особенности реализации

### Парсер ТЗ
- Автоматическое определение формата документа
- Сохранение иерархии разделов
- Обработка различных типов списков

### Векторизатор
- 10-мерные векторы для каждого блока текста
- Словарь тематических слов для улучшения качества
- Косинусное сходство для сравнения векторов

### Интерфейс
- Приятная кофейная цветовая схема
- Удобное расположение элементов управления
- Информативные сообщения о статусе операций

## Примеры использования

1. **Поиск информации об оплате**: "Как оплатить заказ?"
2. **Поиск требований**: "Какие требования к интерфейсу?"
3. **Поиск технических деталей**: "Как работает интеграция с API?"

## Форматы входных данных

Программа поддерживает следующие форматы текста:

### Markdown:
```
# Основные требования
## Функциональные требования
- Оплата по QR-коду
- Уведомления в Telegram
```

### Простой формат:
```
Раздел|Содержание
Требования к оплате|Должна быть поддержка QR-кодов
Интерфейс|Минималистичный дизайн
```
